# Manifest and lock

## agent-skills.json

Project-level declarative dependencies, similar to `package.json`:

```json
{
  "agentSkills": {
    "dependencies": {
      "remote-skill": "github:owner/repo@main#path/to/skill",
      "team-skill": "file:./.skillctl/skills/team-skill"
    }
  }
}
```

Skill name keys are canonical lowercase-hyphen names. Commit the manifest and any project-vendored `.skillctl/skills/` content.

## agent-skills.lock

Reproducible YAML lock, similar to `pnpm-lock.yaml`. Key fields per skill:

- `specifier` — mirrors the portable manifest request.
- `resolved` — immutable GitHub commit or exact npm version; local entries use their vendored `file:` specifier.
- `integrity` — `sha256:` tree hash of the installed copy.
- `canonicalPath` — `.skillctl/skills/<name>` for project entries or `~/.skillctl/skills/<name>` for explicit global entries.
- `provenance` — source metadata such as `github`, `local`, `npm`, or migration origin.

GitHub/skills.sh provenance includes `commit` and `requestedRef`; npm provenance includes `version`, `tarballUrl`, and `tarballHash`. The additive schema remains `lockfileVersion: '1.0'`.

`install --frozen` may materialize a missing or corrupt store from immutable remote fields, but never changes the lock. It rejects missing entries, manifest mismatches, mutable legacy resolutions, and integrity mismatches. Local/imported content must be committed under `.skillctl/skills/` because it has no remote source to restore from.

Commit manifest, lock, and required vendored skills for team workflows. Machine-local agent targets such as `.claude/skills` and `.grok/skills` are generated by `sync` and should not be committed.
